# sitkaz.kz — как встроить материал книги в приложение

Аналитическая записка. Ничего в коде не менялось — это план для ознакомления.

---

## 1. Главный вывод

У книги есть готовая авторская структура, которая ложится на приложение
лучше, чем наши нынешние фразы-заготовки. Не нужно придумывать модули и уроки —
они уже заданы: 14 глав, внутри пронумерованные подглавы (1.1, 1.2, 2.1 …).
Это снимает всю работу по проектированию оглавления курса.

Но есть принципиальный момент, который надо решить до наполнения (см. раздел 5
о правах). От него зависит, переносим ли мы текст как есть или используем книгу
как каркас, наполняя своими формулировками.

---

## 2. Что реально лежит в каждой главе

Разобрав главы I–VI, я вижу пять разных типов учебных единиц. Сейчас в
приложении есть только один из них (плоские «фразы»). Вот все пять:

1. **Термины и понятия.** Короткие казахские слова с точным значением
   (термины родства, названия обычаев, ключевые понятия). → это словарь.

2. **Пословицы (мақал-мәтел).** Казахский оригинал + смысл/эквивалент.
   Их в книге очень много — это её ядро и сильнейшая сторона.

3. **Фразеологизмы и языковые штампы.** Готовые выражения под ситуацию:
   благопожелания, устойчивые реакции, формулы вежливости. → это то, что
   ближе всего к нынешним «ситуативным фразам» приложения.

4. **Устойчивые словосочетания.** Например, целые таблицы сочетаний вокруг
   одного слова (со значениями). → удобный отдельный формат «слово + сочетания».

5. **Культурные заметки.** 2–3 абзаца объяснения «почему так у казахов»
   (устройство родословной, жузы, гостеприимство и т.п.). → короткий
   экран-чтение перед изучением фраз. Именно это делает курс «ситуативным».

## 3. Как это ложится на структуру бота

### Модули = главы (их 14)

Сейчас 3 модуля-заготовки. Заменяем на реальные главы книги:

| № | Тема (модуль) |
|---|---|
| I | Происхождение (ТЕК) |
| II | Семья (ҮЙ-ІШІ) |
| III | Родственники (ТУҒАН-ТУЫСҚАН) |
| IV | Характер (МІНЕЗ) |
| V | Наследие (МҰРА) |
| VI | Мировоззрение (КӨЗҚАРАС) |
| VII | Этикет (ЖӨН БІЛУ) |
| VIII | Женитьба (ҮЙЛЕНУ) |
| IX | Похороны (ЖЕРЛЕУ) |
| X | Пожелания (БАТА-ТІЛЕК) |
| XI | Пословицы (МАҚАЛ) |
| XII | Фразеологизмы |
| XIII | Имена (АТТАР) |
| XIV | Поговорки о родах |

### Уроки = подглавы

Они уже пронумерованы в книге (например, глава II: 2.1 дедушка-бабушка,
2.2 отец-мать, 2.3 супруги, 2.4 дети). Берём прямо эту разбивку.

### Единица контента получает тип

Сейчас урок — плоский список `phrases`. Предлагаю каждой единице дать поле
`type`, а уроку — опциональную вступительную заметку:

```
lesson = {
  id, module,
  title,            // казахский заголовок урока
  ru, en,           // перевод названия
  note: { ru, en }, // короткая культурная заметка (2–3 абзаца)
  items: [
    { type: "term",    kk, ru, en, tr },  // термин-словарь
    { type: "proverb", kk, ru, en, tr },  // пословица + смысл
    { type: "phrase",  kk, ru, en, tr },  // ситуативный штамп
    { type: "colloc",  kk, ru, en, tr },  // устойчивое сочетание
  ]
}
```

Это обратно совместимо: старые фразы просто получают `type: "phrase"`.

## 4. Что добавить в интерфейсе

Небольшие расширения нынешнего пошагового тренажёра (Изучение → Практика → Экзамен):

- **Экран-заметка** в начале урока: 2–3 абзаца контекста перед фразами.
  Барс-маскот в роли рассказчика. Это и есть «ситуативность» —
  сначала «почему так», потом сами выражения.

- **Карточка «пословица»** — визуально отличается от обычной фразы
  (значок, подпись «мақал»), оригинал крупно + смысл. В квизе для пословиц —
  упражнение «подбери смысл».

- **Карточка «термин»** — казахское слово + короткое определение;
  в квизе «что означает».

- **Формат «слово и его сочетания»** — для таблиц вокруг одного слова:
  один экран-хаб со списком сочетаний и переводами.

Ни один существующий экран при этом не ломается — это добавления поверх.

---

## 5. Права (решить до наполнения)

Книга помечена запретом на воспроизведение без письменного разрешения автора.
Приложение публичное (живёт на Vercel), поэтому дословный перенос пословиц,
фраз и авторских переводов в бот = воспроизведение. Два чистых пути:

- **A. Получить разрешение** Каната Тасибекова на использование материала.
  Тогда переносим содержимое как есть — это лучший вариант по качеству.

- **B. Книга как каркас.** Используем её структуру (главы, подглавы, набор тем
  и понятий — это факты, не объект охраны), а конкретные фразы, пословицы и
  пояснения формулируем самостоятельно / из открытых источников.

Оба пути совместимы с планом выше. Разница только в том, откуда берётся текст
внутри слотов.

---

## 6. Предлагаемый порядок работ

1. Расширить модель данных (`type` + `note`) и добавить новые типы карточек
   и экран-заметку — на 1–2 примерах-заглушках, чтобы увидеть, как урок из
   книги выглядит в боте. Это чистая разработка, без контента.
2. Определиться с правами (путь A или B).
3. Наполнить главы контентом по выбранному пути, поглавно.
4. Постепенно заменить нынешние фразы-заготовки на реальную структуру.

Кидать остальные главы для проектирования уже не нужно — структуру я вижу
целиком. Главы понадобятся только на этапе наполнения (шаг 3).
